## Design Context

### Users
ServiceNow practitioners (admins, developers, solution architects) using this as a self-hosted project management tool for ServiceNow workflows. Power users who live in dense tooling. Context: work environment, desktop-primary, sustained sessions.

### Brand Personality
Precise. Grounded. Purposeful. ServiceNow green (#2DB55D) brand accent over a deep teal/navy dark-mode base.

### Aesthetic Direction
Dark mode primary. Deep teal/navy layered system (#042E43 page → #174D6D cards) with ServiceNow green accents. Cal Sans UI for body, Cal Sans Heading for display. Tone: professional project tool — not playful, not austere. Elevated card surfaces that clearly float above their containers.

Anti-references: generic SaaS dashboards, cyan-on-dark AI aesthetics, glassmorphism for its own sake.

### Design Principles
1. **Elevation through contrast** — cards must visually float above columns; use background, border, and shadow in combination
2. **Color is semantic** — use `text-warning`/`text-info`/`text-destructive` for standalone icons; `*-foreground` variants are only for text/icons ON colored backgrounds
3. **Interaction states must be complete** — default, hover, active, selected, dragging; no state left unstyled
4. **Density over decoration** — information density is a feature; don't pad or space-out unnecessarily
5. **Consistency over novelty** — follow established patterns; only deviate when there's a clear functional reason
